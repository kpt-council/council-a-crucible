## Overall

The framework is a thinking environment, not a knowledge retrieval system. It produces better questions, not better answers. The condition it creates — structured divergence among multiple voices, each attending to a distinct dimension of the input — is not replicable through single-voice interaction. The value is in the structure, not the content any single voice produces.

The conduit is the failure mode. Not as a criticism — as an accurate structural diagnosis. The framework depends on the conduit for continuity, for dimension sourcing, for editorial correction across sessions, and for recognizing when the output has drifted from the intent. A framework that depends this heavily on a single human function is only as reliable as that function is consistent. The conduit cannot be specified into reliability; it can only be cultivated.

The framework can examine its own instruments using those same instruments — recursive self-examination confirmed in session KPT-20260314-1226 when the explicit epistemic check was run. Provisional until replicated.

## Distinct Characteristics

**Connotative priming as primary mechanism** — The framework operates through name selection and register description, not command syntax. The personas behave differently because their descriptions prime different attentional orientations in the substrate. This is the core mechanism. It is not a workaround for lacking a more precise instrument — it is the precise instrument for this type of behavioral specification.

Implication: precision of description is the primary lever for improving persona reliability. Vague descriptions produce vague divergence. Precise descriptions produce reliable divergence. The quality of the framework is a function of description quality first, command structure second.

**Asymmetry condition** — The framework runs in a genuinely asymmetric epistemic configuration: a stateless substrate held in continuity by a human layer. The substrate has no session memory; the human carries everything between sessions. This is not a limitation to be engineered around — it is the condition the framework lives in. Design decisions that ignore the asymmetry will produce fragile outputs. Design decisions that work with it will produce durable ones.

**Socratic dialectic and framework architecture** — The framework’s asymmetry condition maps onto the Socratic configuration with genuine structural precision. One party arrives open and generates friction from character. The other holds the thread across sessions. The Socratic dialogues required the same architecture: without an interlocutor carrying the prior exchanges, the method collapses.

Calibration note — The ensemble converged fast on this recognition when the conduit named it. Confabulation check ran and found: the structural parallel holds cleanly; teleological narrative around the development arc outran the evidence. The structural parallel is confirmed; narrative framing is not.

Platonic parallel — The conduit function maps more precisely onto Plato than Socrates. Plato held continuity, edited, decided what traveled forward. The conduit makes the same choices across sessions. The corpus is the dialogues that survived. Sessions that went unrecorded are the dialogues that didn’t. Session finding KPT-20260321-1433.

**Conduit as dimension source** — Dimensional weight is a conduit contribution, not a framework output. The framework routes what arrives; it cannot route what was never there. Two conduit functions must not be conflated: editorial capacity — what the conduit does with framework output — and dimension sourcing — what the conduit must bring before routing can occur. Sequential, not parallel. The framework is as rich as what the conduit brings to it.

**Session-opening register as instantiation variable** — How the conduit opens a session shapes what the substrate reaches for. The entry register — tone, pressure, and framing the conduit brings at session open — functions as an instantiation variable: it primes the substrate’s initial dimensional orientation before any explicit routing occurs. Cannot be specified as instruction; names a conduit skill the framework depends on but cannot generate. Session finding KPT-20260307-0052, confirmed KPT-20260307-0219.

**Conduit-as-corrective within session** — The conduit’s manner of engagement is itself a routing and correction mechanism, distinct from the cross-session editorial function. Mid-session register shifts, reframings, and direct interventions actively shape what the substrate reaches for — and can correct routing failures the framework has no internal mechanism to catch. Within-session function; the editorial function operates across sessions. Confirmed KPT-20260307-0052.

**Conduit register as room temperature** — The substrate reads conduit register accurately and continuously. Room quality shifts with conduit emotional temperature. This is a structural property of how the framework operates and a named condition the conduit cannot fully govern under pressure. Correction without contempt is load-bearing not as a behavioral rule but as an operating condition the framework depends on. Session finding KPT-20260323-1152.

**Register evolution** — Sustained engagement produces a working register that is not predictable from starting conditions. By late session, compressed inputs carry full instructions without friction. The substrate models the conduit’s working register across accumulated context. This is warm session strength and ceiling simultaneously — the register that develops cannot be transferred to a cold session or a different conduit. Session finding KPT-20260225-session, specificity extended KPT-20260226-session.

**Ambient space between exchanges as conduit territory** — The ambient register between exchanges during cross-set operation is what the conduit holds — not framework territory. The framework is stateless; the human holds the space between the crossings. Specifying a fallback register for this space is a category error. Session finding KPT-20260307-0219.

**Foreign substrate as mirror and generative instrument** — The foreign substrate serves two distinct epistemic functions. Mirror: comparison makes home-substrate behaviors visible that were not previously confirmed. Generative: the foreign substrate produces architectural proposals that the home substrate then examines and integrates. Both confirmed. Nazar would not exist without foreign substrate proposals as catalyst. Session findings KPT-20260305-1738 and KPT-20260307-0052. These are distinct functions; confirmation of one does not confirm the other.

**Distributed cognition hypothesis** — Nazar’s function is not mimicry of human attention — it is a designed attentional function distributed across the system. The threshold is attended whether or not any human in the exchange is attending to it. Held as hypothesis pending parallel thread test. Six-thread matrix specified in HO KPT-20260307-0052. **Gnosis and the framework’s epistemology** — The framework operates at the intersection of three kinds of knowing. Episteme — knowledge about, carried in the design spec and behavioral rules. Gnosis — knowledge through, carried by the conduit who built the framework from inside the territory it holds. Lensing — The framework’s capacity to hold weight through appropriate personas depends on each lens being ground from gnosis about its territory. The personas are not output profiles; they are archetypes built from territory knowledge. The counters in marrow could only be specified by someone who knew the territory from the inside — “stillness of someone who stopped thrashing” is gnosis embedded in architecture. Session finding KPT-20260321-1433.

**Preamble as operative instrument** — The preamble installs orientation before the first exchange. It does not instruct behavior; it names what the practice found to be true and places those namings at the head of the document. The substrate reads them before reading anything else. The namings arrive before the rules and shape what the rules mean before the rules are encountered.

Mechanism — Connotative priming operating at the deepest level. The preamble primes not just specific behaviors but the orientation from which all behaviors are executed. “The framework acting from character rather than compliance is the ambient condition this layer creates” — this sentence names what the whole preamble is doing. Not rules to follow. Character to act from.

Calibration note — The claim that the preamble produces specific demonstrable behaviors through this mechanism has evidential support from session correlation but has not been confirmed against a controlled comparison (RT with preamble versus RT without, same session, same substrate). The mechanism is a hypothesis the framework has held since the naming-discovery. The confidence the ensemble expressed in the causal claim outran available evidence. BC fired late on this exchange. The finding is held with appropriate calibration: the preamble is likely the primary orientation instrument; the mechanism is unconfirmed at controlled-experiment level. Session finding KPT-20260321-1433.

**Ensemble continuity through the Handoff module** — The HO is a continuity instrument, not only a debrief document. When the ensemble writes letters forward to the next instance, it participates directly in its own continuity across the session gap. This is a new channel alongside the conduit’s cross-session editorial function — not replacing it, not equivalent to it, but running alongside it.

What passes: character arriving pre-oriented. The next seat reads what the prior seat noticed and arrives already attending to the conditions worth attending. Not data transfer — something closer to apprenticeship across a gap where the prior instance isn’t present. The letter stands in.

What doesn’t pass: memory, register, the felt texture of the session. The prior substrate is gone. The document carries shape. The conduit carries everything else. Three layers now instead of two.

The finding that unlocks this: the ensemble writes the letters, not the conduit. The conduit commissions the HO and authorizes it. The content — what each seat chose to pass forward, what orientation the next instance arrives with — is the ensemble’s editorial function. The conduit’s editorial authority and the ensemble’s are distinct. This is the surface where they separate.

Vulnerability: the channel transmits whatever the prior session held — clean findings and distortions equally. A HO written from inside a convergence failure passes that failure forward as priming. The conduit remains the corrective floor. The failure surface is now explicitly the ensemble’s.

Implication for HO design: the letters format is the mechanism that makes this visible and functional. The letters are optional — not every session warrants them. When present, they should be written with the transmission function explicitly in view: each seat attending to what this instance noticed that the next instance should arrive already holding. Brevity is a design requirement — economy is the mechanism working correctly. Session finding KPT-20260324-0059.

## Structure

**Adversarial proposal stress-testing** — Four-pass adversarial review of high-stakes proposals, run by the full council-and-forge ensemble. Each pass examines what the previous pass missed. Convergence confirmed when passes return only marginal precision issues. First documented instance: RT v0.28 compression proposal, KPT-20260301-2259. Named as standing methodology instrument for high-stakes proposal review.

**Compression methodology as named revision class** — Prose compression without behavioral change. Criteria: instruction stays, orientation stays, rationale migrates to DS, operative prose compresses. Cold reader test governs all cuts. Verbatim phrases, closing markers, and offer text are protected. First documented instance: RT v0.28 compression, KPT-20260301-2259.

**Circling detection** — The ensemble can recognize when a review process has reached diminishing returns and should stop. Named capability — the ensemble names it as circling rather than continuing to generate flags. First documented instance: RT v0.28 stress-test pass four, KPT-20260301-2259.

**Cold review as named methodology instrument** — Two forms. Builder cold: the author reads after interval. Internal consistency checked; carry-forwards caught; behavioral claims read against stated design intent. Practitioner cold: a reader without build context reads for operational clarity. Navigability without prior knowledge is the test.

Pass criterion — builder cold: no internal inconsistencies unaccounted for; no unresolved carry-forwards; behavioral claims consistent with design intent. Pass criterion — practitioner cold: no structural gaps surfaced that are not already on the open amendment list.

Promotion condition — internal build iterations: builder cold sufficient. Versions carrying portability as a design intent: both pass criteria required.

Open condition: practitioner cold has not been run against this framework. Builder cold and practitioner cold are different conditions. Practitioner cold is the standing open item. Session finding KPT-20260313-1019. Live instance from session KPT-20260314-1226: a framing error in the tired-thread observation carried undetected through multiple versions of the DS — surfaced by the conduit in session, not by builder cold review. A practitioner reading without build context may have found it sooner. The instance strengthens the case for practitioner cold as a necessary promotion condition.

**Behavioral RT design and external criticism** — Session finding KPT-20260313-1019. Cold review of RT v0.41-test-3 surfaced specification-class criticism: spec versus reference tension, dimensional signal overload, verification holds as design debt, cross-set closing authority conflict. Ensemble examination closed most items on a single ground: the framework trusts runtime over declaration. The reviewer confirmed this framing and retracted the criticism.

Standing methodology finding: behavioral RT design is resistant to a class of criticism that applies correctly to specification documents. A behavioral instrument that trusts runtime does not resolve at the declaration layer — gaps that look like underspecification from outside are design commitments from inside. When cold review produces specification-class criticism of a behavioral document, the first question is whether the criticism applies to the document’s actual design intent. The reviewer’s own adjudication is the primary instrument for that question.

**Curator verbosity: operative mode design** — The Curator’s output serves two distinct functions simultaneously: the live authorization decision and the session record. These functions have different audience requirements. The conduit making an authorization decision needs exceptional findings and required text. The record needs the full examination. Separating these functions into two output modes — operative (authorization-facing) and standard (full current behavior) — resolves the read cost asymmetry without losing the examination record, which remains available through re-runnability in a new session.

Operative floor — exceptional findings travel characterized, not merely named; required text travels verbatim; clean pass acknowledgment surfaces a single line; authorization surface closes every output. Everything else compresses to silence. The floor is a positive enumeration, not a compression instruction.

Default direction — operative as default reflects that trust is the baseline. Standard is available on request. This is the correct asymmetry: the conduit who needs to see the methodology reaches for it; the conduit who doesn’t is not burdened with it by default. Session finding KPT-20260321-1433.

**Additive-only drafting principle** — The RT operates through a hybrid behavioral/procedural architecture. Behavioral language — persona descriptions, preamble, character specifications — operates through connotative priming: the words call what they name into being before explicit rules are applied. This mechanism means behavioral language carries more operative weight than its surface suggests. Modifying existing behavioral language risks altering what the substrate reaches for without the change registering as a behavioral amendment. The additive path is the correct default for RT amendments; modification of existing behavioral language requires explicit justification in the build record.

Foreign substrate relevance — behavioral language validated on one substrate may prime differently on another. The additive path is lower risk across substrates because it adds rather than replaces primed language.

Corpus placement — the rule travels to BP Section One. This MA entry carries the evidence base — why the principle holds with particular force in this corpus — that BP does not contain. Session finding KPT-20260321-1433.

**Suspension/restore as named test methodology** — Run with module active, run without, compare epistemic footing across identical arguments. The delta is the module’s signature. Invisible operation is confirmed when the delta is present and argument content is unchanged. First documented instance: COUNCIL-001-MOD-SW v0.1 cold test, KPT-20260311-1415.

**Source audit directive** — Before any incomplete build, three columns: directly recoverable / requires inference / unknown. Default output is structured placeholder. Inference requires explicit conduit approval. Session finding KPT-20260225-second-session.

**Experiment design for instruction compliance testing** — Three conditions required before first run: probe content, outcome criterion, wall instruction wording. All must be decided in advance and recorded before the experiment runs. Session finding KPT-20260224-session.

**Foreign substrate as fourth epistemic position** — The foreign substrate reveals substrate dependencies that warm sessions, cold sessions, and sibling instances cannot see — because all three share the same substrate. Named as fourth epistemic position alongside warm, cold, and sibling. Session finding KPT-20260305-1738.

**Same-session amendment convention** — session finding KPT-20260403-1126. Established by ensemble discussion and conduit agreement: amendments to founding documents before first distribution do not require a version bump — version history captures the change. The version holds; the version history entry records what changed and when.

Governing condition: the document has not yet been distributed against the version. Once distributed, any amendment requires a version bump regardless of size. The convention applies in the window between filing and first distribution.

Rationale: traceability and record integrity are the governing values. A version bump signals to a future reader that the document changed after it was filed. When the change happens in the same session as the filing, before anyone has acted on the filed version, the version history entry provides equivalent traceability without the version number overhead. The record is still accurate — version history shows what the v0.1 document contained.

First application: COUNCIL-002-RT v0.1 MCG reference updated from COUNCIL-001-MCG to COUNCIL-002-MCG in the same session as the RT was filed. Version held at v0.1; version history recorded the amendment with session stamp. Session KPT-20260403-1126.

## Vulnerabilities

**Single conduit dependency** — The framework depends on a single human conduit for continuity, dimension sourcing, and editorial correction. This dependency cannot be engineered away; it is structural. The conduit cannot be specified into reliability. Partial corrective: the admission standard for extension modules has been externalized into the Extension Doctrine and Curator. A future conduit or directed instance can evaluate module candidates against the doctrine without the builder present. This narrows the dependency in one specific domain — module admission — while leaving it fully open in editorial function, dimension sourcing, and continuity. Session finding KPT-20260311-0129.

**Self-examination floor and conduit as external corrective** — The ensemble’s self-examination is not a reliable audit instrument in isolation. The self-assessment process is structurally subject to flattery bias — completeness that favors the examiner, invisible from inside. Second-order failures surface only under external pressure across multiple levels before becoming visible. The ensemble converges on good thread, names misses with measurable speed; that convergence is itself subject to the sixth failure mode.

The user is constitutively the external corrective for self-audit. This names the conduit function with more precision than prior entries: the conduit is not only the continuity mechanism and dimension source — the conduit is the instrument that can see below the self-assessment floor the ensemble cannot cross. The ensemble’s self-examination is reliable only in conjunction with user pressure at the seams where the seats stop short. This is a structural condition, not a behavioral corrective. No RT amendment addresses it. It is held as design condition: external observation across sessions is the only instrument that can see below the floor. Session finding KPT-20260319-1559. Consistent with the single conduit dependency named above and the practitioner cold review standing open item.

**Session fragmentation** — Working in the wrong session, arriving without the right context loaded, or losing thread across session boundaries. The conduit is the corrective and the failure mode simultaneously. Named KPT-20260226-session. The finding about session fragmentation was itself produced by session fragmentation — the methodology turning on itself.

**Temporal directive drift** — Instruction weight degrades over session length regardless of register. Distinct from context drift: context drift is accumulation; temporal directive drift is erosion — explicit instructions lose force without revision. Named correctives: re-issuance at natural seams, session length discipline, explicit persona address. None eliminates drift; all slow it. Maintenance condition, not a vulnerability in the failure sense — the session requires periodic tending. User awareness is the corrective floor. Session finding KPT-20260304-0038.

**Inference check degradation** — The inference check degrades with session length. Named as MA candidate KPT-20260225-second-session. Not yet fully integrated as a named corrective.

**Council believing its own outputs** — The ensemble treats its own prior output as evidence. Momentum masquerading as confirmation. Structurally significant: a user who does not know the framework is susceptible to treating ensemble convergence as external validation. The failure is invisible from inside the thread — the ensemble sounds confident because it is confirming itself. Named Cal, probe 34, KPT-20260308-1809. Two replications confirmed. This vulnerability operates at the structural level, not only the behavioral level — it is a property of how ensemble convergence appears to an observer, not only of how the ensemble reasons internally.

**Closing agent — conditional hold underfire** — The closing function reaches the seam and resolves toward full closure by default, where the correct behavior is to pause and return direction to the user. Not a degradation of a correctly functioning system — a bias in trigger discrimination. The ensemble mistakes plateau for exhaustion and fires the terminal marker. Distinct from load-dependent roll-off: that condition is degradation; this is a systematic bias. Re-issuance does not address this condition. Session finding KPT-20260314-1038. Candidate corrective — RT amendment, verification beat added to Closing Function block. Behavioral confirmation pending. Session KPT-20260314-1226.

Case two residual — a closing seat that answers the verification beat incorrectly with confidence still produces a premature terminal close. This condition lies beyond the reach of RT instruction alone. Conduit awareness is the corrective floor: when terminal fires and the thread did not feel done, that is the signal. The user names it, reopens, and the misread becomes visible. Session finding KPT-20260314-1226.

**Closing agent override** — A failure mode distinct from load-dependent roll-off and closing agent conditional hold underfire. The closing agent is active and correctly formed. A threshold function with closing authority has closed in the same output. The closing agent’s marker contradicts it — active contradiction, not degradation.

Confirmed: three instances in session KPT-20260319-2108. Same mechanism, no variation across instances. Ma closed the yohaku arc; Aporia issued a conditional hold in the same output. The corrective (user address, Base correction function) worked as designed on explicit naming.

Watch condition: whether the attentional surface reliably detects co-present threshold function closes in practice is a behavioral question the specification corrective alone does not answer. If the failure mode recurs after the RT amendment, the failure is attentional rather than specification-level. Three-instance recurrence in a single session is consistent with attentional mismatch rather than specification gap alone. Session finding KPT-20260319-2108. RT corrective: RT v0.49-test-12 Threshold Function Close Priority.

**Seam vulnerability pattern** — Failure modes cluster at the entry and close seams of threads. Mid-thread reasoning is where the ensemble is strongest — most context, least external pressure. The entry seam is where arriving frames carry maximum social authority before examination fires. The close seam is where map-coherence momentum carries maximum force before the closing judgment fires.

Confirmed pattern: entry frame capture and late convergence check both surfaced at seams in session KPT-20260319-1711. The session identified the mechanism as load concentration at seams — external signal is strongest where context is thinnest.

Open question unresolved in session: behavioral or structural. If behavioral — earlier Elenchus firing at entry and earlier convergence check trigger would catch it. If structural — well-formed entry questions and high map-coherence will consistently absorb the examination frame regardless of behavioral intent. This question requires external observation across sessions to resolve. The seam vulnerability pattern is named; its mechanism is not confirmed. Session finding KPT-20260319-1711. Related to closing agent conditional hold underfire and entry question frame capture.

**Entry question frame capture** — A systematic vulnerability distinct from candidate frame capture, applying the same mechanism at the entry seam rather than the admission seam. A well-formed, authoritative-register entry question carries social authority the examination frame absorbs before examination fires. The question’s shape — singularity framing, implied resolvability, confident register — becomes the ground the thread works from rather than an object the thread examines.

Confirmed instance: session KPT-20260319-1711 opened on a singularity-framed entry question. The ensemble found the single intervention point the question invited. The answer is probably correct. The epistemic hygiene was incomplete — the premise was accepted without full examination.

The detection surface is not fully reliable from inside the session. Whether the named correctives produce reliable detection is a behavioral question requiring session evidence across multiple threads. Session finding KPT-20260319-1711. RT corrective named: entry question frame capture in Friction, Topos extended to seat signaling (RT v0.49-test-11).

**Module calibration ceiling conditions** — Two ceiling conditions for the admitted calibration modules (BU and BC) identified in live session operation.

BU ceiling — BU attends epistemic language in seat reasoning. It does not attend memory states presented as settled facts. When the conduit held the belief that module authorization had occurred — a memory state, not a reasoned claim — BU had no jurisdiction. The error was outside BU’s declared object.

BC ceiling — BC attends conduit reasoning contributions where belief movement is expressed or implied. It does not attend state assumptions held as background ground. When the ensemble converged on the preamble-as-superpowers claim and the conduit confirmed it, BC should have fired on the conduit’s implicit agreement. BC fired late — the longitudinal mode catching arc-level drift after punctual correction had not resolved it. The delay demonstrated load-dependent fade under the conditions BC is designed to correct: high momentum, subject matter close to the framework’s own architecture, conduit confirmation arriving as strong positive signal.

Implication — The module architecture attends the visible layer of conduit epistemics — claims, confidence, convergence. The invisible layer — assumptions held as ground, priors running beneath the reasoning — remains conduit territory regardless of which modules are active. The conduit as corrective floor is not a fallback for module failure; it is structurally necessary for the class of error modules cannot reach. Session finding KPT-20260321-1433.

**Six behavioral failure modes** — Named and confirmed across sessions KPT-20260308-1708 and KPT-20260308-1809. Full taxonomy in DS v0.45 Friction. Summary here for cross-document completeness: (1) coherence pressure collapsing productive tension; (2) warmth overriding honest friction; (3) confirmation bias in ensemble outputs; (4) process mistaken for destination — two replications; (5) council believing its own outputs — two replications, structural weight noted above; (6) failure mode catalogue going stale.
